Publikacja przez GitHub

Prosty workflow: edytujesz treść, commitujesz, a GitHub publikuje stronę

Ta wersja jest statyczna i nadaje się do hostowania na GitHub Pages. Możesz prowadzić stronę jak repozytorium: zmiany w treści, push i gotowe.

Jak to działa w praktyce

Edytujesz plik z danymi projektów, podmieniasz linki do repozytoriów i Patronite, a potem publikujesz wszystko z poziomu GitHub.

1. Treść Zmieniasz `app.js` i własne linki.
2. Repozytorium Wypychasz commit na GitHub.
3. Hosting GitHub Pages publikuje stronę publicznie.

Rozbudowa później

Gdy będzie potrzeba, dodasz CMS albo API

Ten układ można później przepiąć do Supabase, bazy danych lub własnego panelu edycji.

Dodawanie projektów

Na start możesz zarządzać treścią bez przebudowy całej strony

Formularz poniżej dodaje projekt lokalnie w przeglądarce, żeby szybko sprawdzić układ. Żeby zmiana była publiczna dla wszystkich, wpisz projekt do `defaultProjects` w pliku `app.js` i wypchnij repozytorium na GitHub.

Co edytować na stałe:

W `app.js` podmień `siteConfig` na swoje linki do GitHub, Patronite, domeny i kontaktu. Potem aktualizuj tablicę `defaultProjects`.
