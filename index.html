Publikacja przez GitHub

Prosty workflow: edytujesz treść, commitujesz, a GitHub publikuje stronę

Ta wersja jest statyczna i nadaje się do hostowania na GitHub Pages. Możesz prowadzić stronę jak repozytorium: zmiany w treści, push i gotowe.

Jak to działa w praktyce

Edytujesz plik z danymi projektów, podmieniasz linki do repozytoriów i Patronite, a potem publikujesz wszystko z poziomu GitHub.

1. Treść Zmieniasz `app.js` i własne linki.
2. Repozytorium Wypychasz commit na GitHub.
3. Hosting GitHub Pages publikuje stronę publicznie.

Dodawanie projektów

Nowe projekty możesz dodawać tylko jako właściciel repozytorium

Usunąłem publiczny formularz z frontu, żeby nikt odwiedzający stronę nie mógł sam dodać wpisu. Od teraz projekty dopisujesz wyłącznie przez GitHub, czyli tylko z Twoim dostępem do repo `grzecho322-lgtm/GP-Electronics`.

Bezpieczny model właścicielski:

Strona pozostaje statyczna i bez backendu, ale tylko Ty możesz dodawać projekty, bo publikacja odbywa się przez edycję pliku `app.js` w repozytorium, do którego masz uprawnienia zapisu.

Panel właściciela

Dodawanie projektów tylko przez GitHub

Najprostsza i bezpieczna droga to edycja pliku z danymi projektów bezpośrednio w repo. Po commicie GitHub Pages automatycznie pokaże nową zawartość na stronie.

Edytuj `app.js` na GitHub Otwórz repo
  1. Otwórz plik `app.js` w repozytorium.
  2. Dopisz nowy obiekt do tablicy `defaultProjects`.
  3. Zapisz commit na gałęzi `main` albo przez własny branch.
  4. Po publikacji GitHub Pages nowy projekt pojawi się publicznie na stronie.
Ważne:

Tylko konto z uprawnieniami do repo może dodać lub zmienić projekt, więc odwiedzający stronę nie mają żadnej publicznej ścieżki edycji treści.
